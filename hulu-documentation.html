Hulu — ሁሉ OS: Personal Life Operating System

Hulu — ሁሉ (meaning "Everything" or "All" in Amharic) is a premium, high-integrity personal operating workspace. Rooted in Next.js 16 (Turbopack), React 19, Tailwind CSS v4, and Supabase, Hulu transforms standard task tracking into a calm, broadsheet-inspired executive platform. By coupling rigid architectural layouts with a selective crimson accent system, Hulu provides focus, clarity, and visual peace for daily operations.

Flat Aesthetics

Strictly devoid of gradients, round curves, or glowing overlays. Structure is guided by empty space, sharp borders, and fine borders.

Journalistic Typography

Broadsheet-inspired typeface hierarchy. Serif titles (Fraunces/Zemach), structural UI monospacing (Space Grotesk), and reading screens (Public Sans).

Focused Interaction

Exactly one active focus point is highlighted in Crimson/Coral on any page view, aligning visual flow with immediate focus.

Design Philosophy & System Spec

The core aesthetic design guidelines of the Hulu OS.

System Color Tokens

Curated flat color variables designed for reading comfort and high accessibility contrast across themes.

Warm Limestone (Neutral Light)

`#F7F5F2` • Light mode background. Invokes texture of newsprint or physical paper.

Slate Black (Neutral Dark)

`#101112` • Dark mode background. Designed to reduce fatigue in late reviews.

Charcoal Ink (Primary Ink)

`#1A1C1E` • Default text and borders in light mode; secondary contrast in dark mode.

Crimson Accent (Light Primary Action)

`#B8422E` • Guides light mode user focus to primary CTA button, active checkboxes, and select indicators.

Coral Accent (Dark Primary Action)

`#E2583E` • Contrast-adjusted crimson equivalent driving attention in dark mode.

Rules of the Layout Grid

  • The Single-Accent Constraint: Exactly one major interactive CTA is styled in Crimson/Coral per viewport. Side-actions are styled as muted outlines.
  • Structured Empty Space: Margins and empty areas are structural. Grid columns are separated by empty spaces or standard 1px borders.
  • Zero Gradients or Shadows: Avoid Tailwind gradients (`bg-gradient-to-r`) or shadow drop-shadow classes. Borders and backgrounds are strictly solid.
  • Typography Hierarchy: Display titles use `Fraunces` / `Zemach` (Amharic). Structural controls use `Space Grotesk`. Read notes use `Public Sans`.

Data Pipelines & Flows

Flow diagrams and core processing steps driving Hulu OS.

Inbox Capture to Action Flow

When an input is captured, it flows through a sequential pipeline from extraction to persistence:

  1. Ingestion Trigger: User submits text or uploads a photo from [UniversalCaptureModal](file:///home/adonaymick/Documents/Life/src/components/UniversalCaptureModal.tsx).
  2. Resilient Scraping: If the input is a URL, `/api/capture/enrich` uses a custom HTTP client with fallback headers to bypass Cloudflare and parse the page metadata.
  3. AI Structuring: Gemini extracts the core concept. If Gemini is down/limited, local heuristic parsers intercept, scanning regular expressions (like checking `?` to force a `question` category).
  4. Supabase Persistence: The enriched object is stored in `inbox_items` and loaded asynchronously into `DashboardContext` state.
  5. Triage Conversion: Reviewing an item creates a corresponding `tasks` row or `knowledge_items` note, linking the entities together.

AI Research Agent Pipeline

Generating study guides runs sequentially as an asynchronous pipeline with pacing delays:

  1. Source Discovery & Extraction: `/api/research/search` queries Exa API. If Exa is limited, it crawls DuckDuckGo HTML and falls back to Wikipedia API. Raw sources are extracted in parallel using fetch timeout gates to prevent hanging.
  2. Syllabus Outline: `/api/research/outline` consumes research data and structures a course syllabus JSON with modules and descriptions.
  3. Asynchronous Writing: Loops sequentially, writing modules and preserving course/lesson context while using pacing delays and retry backoffs to mitigate Gemini API 429 errors.
  4. Rate-Limiter Interceptor: `useRateLimit` intercepts operations, checking local request logs (limiting to 20 daily calls, 5 RPM, and 250k TPM). It locks inputs if quotas are exceeded.

Soul Blueprint & Oracle Context Pipeline

Maintains an offline context layer compiled from across the system for low-latency AI conversations:

  1. Activity Logging: Mutations across tasks, habits, and journal records insert descriptive rows into `public.activity_events` containing jsonb metadata.
  2. Hourly Re-indexing: A `pg_cron` schedule executes `private.invoke_soul_blueprint_generation()`, triggering the Edge Function `/generate-soul-blueprint` via HTTP POST.
  3. LLM Compilation: The Edge Function fetches all database contexts, formats them to markdown, and compiles a new `soul_blueprint_snapshots` row.
  4. Dynamic Selection: On querying the `/api/oracle/chat` endpoint, the system loads the snapshot and selects relevant sections by matching query keywords against pre-defined tags.
  5. Completion Engine: Selected context is injected as a system prompt to the DeepSeek chat endpoint along with historical memory.

Row-Level Security Policies

Security constraints locking database modifications to authenticated owners.

Enforcing Security at the Database Layer

All tables include `ENABLE ROW LEVEL SECURITY`. Hulu partitions data by validating the active authorization token against the row owner column: 

-- 1. Enable RLS on review entries
ALTER TABLE public.review_entries ENABLE ROW LEVEL SECURITY;

-- 2. Restrict all operations to the authenticated user owning the row
CREATE POLICY "Users can manage their own review entries"
  ON public.review_entries
  FOR ALL
  TO authenticated
  USING (auth.uid() = user_id)
  WITH CHECK (auth.uid() = user_id);

-- 3. Lock down research jobs
ALTER TABLE public.research_jobs ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow authenticated users full access to research_jobs"
  ON public.research_jobs
  FOR ALL
  TO authenticated
  USING (auth.uid() = user_id)
  WITH CHECK (auth.uid() = user_id);

-- 4. Activity log & Soul Blueprint snapshots
ALTER TABLE public.activity_events ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow authenticated users access to own activity_events"
  ON public.activity_events FOR ALL TO authenticated
  USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id);

ALTER TABLE public.soul_blueprint_snapshots ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow authenticated users access to own soul_blueprint_snapshots"
  ON public.soul_blueprint_snapshots FOR ALL TO authenticated
  USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id);

-- 5. Oracle conversations & messages
ALTER TABLE public.oracle_conversations ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow users access to own conversations"
  ON public.oracle_conversations FOR ALL TO authenticated
  USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id);

ALTER TABLE public.oracle_messages ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow users access to own messages"
  ON public.oracle_messages FOR ALL TO authenticated
  USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id);

-- 6. Intake images storage bucket (folder partitioning)
-- Restricts authenticated users to folders matching their auth.uid()
CREATE POLICY "Intake images partition access"
  ON storage.objects FOR ALL TO authenticated
  USING (bucket_id = 'intake-images' AND (storage.foldername(name))[1] = auth.uid()::text)
  WITH CHECK (bucket_id = 'intake-images' AND (storage.foldername(name))[1] = auth.uid()::text);

RUNNING THE DASHBOARD

1. Copy settings template: cp .env.example .env.local

2. Ingest dependencies: npm install

3. Deploy Supabase Edge Functions: supabase functions deploy generate-soul-blueprint --no-verify-jwt

4. Fire up development server: npm run dev

5. Build optimization bundle: npm run build

STAGING & PRODUCTION DEPLOYS

Hulu is pre-configured for automated Vercel deployments:

  1. Push your fork/clone of the dashboard codebase to GitHub.
  2. Import the repository into your Vercel workspace dashboard.
  3. Bind the required Environment variables under project settings.
  4. Builds deploy updates automatically on pushes to the `main` branch.
