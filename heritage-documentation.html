Heritage: A Personal Life Operating System

Heritage (also branded as **Hulu - ሁሉ**, meaning "All" in Amharic) is a full-featured productivity dashboard. Built on next-generation web technologies, it approaches self-management through the lens of **architectural minimalism** and **journalistic gravitas**. Instead of complex, cluttered task grids, Heritage is modeled after quiet galleries and traditional morning broadsheets—inviting focus, clarity, and deep work.

Architectural Minimalism

Stripped of gradients and visual noise. Structure is guided by white space, hard borders, and clear layouts.

Journalistic Gravitas

Editorial typography using serif headers and metadata labels that make your dashboard feel like a publication of your life.

Intentional Accentuation

A strict single-accent system ensuring that exactly one critical interaction point drives your focus per screen.

Design Philosophy & System Spec

The guidelines governing the look and feel of the platform.

Core Colors

We leverage HSL values and curated flat colors. High contrast, warm background undertones, and a singular highlighted action.

Warm Limestone (Neutral Light)

`#F7F5F2` • The light mode background foundation. Easy on the eyes, mimicking physical paper.

Slate Black (Neutral Dark)

`#101112` • The dark mode background foundation. Provides maximum contrast for dark mode.

Charcoal Ink (Primary Text)

`#1A1C1E` • Editorial headlines and body content in light mode. High readability.

Crimson Accent (Light Accent)

`#B8422E` • Active links, primary checkboxes, and load-bearing interaction focus indicators.

Coral Accent (Dark Accent)

`#E2583E` • High visibility interaction driver adapted for dark themes.

The Rules of Heritage

  • The Single-Accent Rule: Exactly one primary CTA button or critical checkpoint should be highlighted in Crimson/Coral on any given page. Do not dilute visual flow with multiple colors.
  • Negative Space is a Feature: White space separates concepts, not heavy horizontal dividers. Avoid cluttered margins.
  • Zero Gradients: The visual system is strictly flat. Color transitions are immediate, and surface textures are matte.
  • Three-Font Scale: Serif for titles (`Fraunces` / `Zemach`), geometric sans for interface details (`Space Grotesk`), and clean sans for reading logs (`Public Sans`).

Features Encyclopedia

Detailed catalog of the platform's core operating engines.

📥 The Inbox Engine

Capture

The Inbox is the absolute beginning of all data in Heritage. It is a digital inbox where you drop unstructured material so that it does not disrupt your current work focus.

  • Item Types: Supports Quick Text, Bookmarked URLs, and Code Snippets.
  • Processing Flow: Allows conversion of inbox items directly into Tasks, Journal reflections, or Course reference materials.
  • Snooze Cycle: Unsorted thoughts can be delayed to reappear at a specific date.

📋 Projects & Tasks

Execute

Heritage splits execution into macro-planning (Projects) and micro-execution (Tasks). Work areas coordinate your client commitments, deadlines, and project values.

  • Areas: Sort projects into categories (Work, Personal, Health, Finance, relationships, etc.).
  • Kanban & Calendar: Interactive view transitions to plan tasks.
  • Smart Pinning & Priority: Pin high-value tasks so they stay locks on the Today Widget.
  • Task Dependencies: Explicit child-parent configurations for subtask pipelines.

⚡ AI Research Engine

Automate

Powered by Gemini models and Exa API, this engine transforms a single keyword or URL topic into a structured course within your Academy.

  • Authoritative Web Search: Crawls references on Exa for valid materials.
  • Syllabus Synthesis: Auto-creates course outlines, modules, and lessons.

Data Flows & System Architecture

How data moves between capture, automated research, and core database components.

System Pipeline Visualization

Unsorted Inbox capturing channels into three core operations. Concurrently, the AI research engine acts on Academy entities to generate study courses, syllabi, notes, and lessons.

Database Schema

Heritage is fully persistent, backed by a relational PostgreSQL database on **Supabase**. User information is isolated at the row-level using PostgreSQL policies, ensuring that each user has access only to their specific records.

Primary Relational Tables

`inbox_items` Captures text, link context, snoozed timers, and processed state. `projects` Holds high-level projects, target area, value metrics, client details. `tasks` Task rows referencing a project. Stores priority, dependencies, recurrences. `habits` Tracks user habits, unit types, numerical goals, and status. `habit_records` Log table containing daily check-in marks and values. `journal_entries` Keyed by date. Contains reflections, intentions, mood value, and notes. `courses` Root containers for structured study subjects. `course_modules` Course chapters containing markdown notes. `lessons` Individual readings or URLs linked to a module. `flashcards` Questions, answers, and SRS parameters for spaced repetition.

Row-Level Security (RLS)

Every table in the database is protected by Row Level Security (RLS). This ensures that database interactions (SELECT, INSERT, UPDATE, DELETE) are restricted to the authenticated user.

Security Hardening Statement

As of the latest migration, policies are locked down so that `auth.uid()` checks are verified at the db layer on every query. Access tokens are generated on the client through the Supabase client wrapper. If a session is missing or expired, the `AuthGuard` component blocks the app, redirecting to the secure auth portal. 

-- Enforcing user ownership on Tasks
ALTER TABLE public.tasks ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can manage their own tasks" 
  ON public.tasks
  FOR ALL
  TO authenticated
  USING (auth.uid() = user_id)
  WITH CHECK (auth.uid() = user_id);

Technical Setup & Deployment Guide

Get the Heritage platform running on your local machine and deploy it to staging/production.

Environment Variables

Environment configuration resides in a `.env.local` file at the root. Duplicate `.env.example` to start: 

# Core Supabase Connectivity
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key-string

# Gemini models configuration (AI Outlines, Notes writing, Q&A)
GEMINI_API_KEY=your-gemini-api-key

# Exa Web Search Integration (Web sources crawling)
EXA_API_KEY=your-exa-api-key

# Pushover Integration (Optional notifications)
PUSHOVER_USER_KEY=your-pushover-user-key
PUSHOVER_API_TOKEN=your-pushover-api-token
DASHBOARD_URL=http://localhost:3000

Prerequisites & Execution

To launch the platform locally, verify your development runtime supports Node.js 20 or higher.

DEVELOPMENT COMMANDS

1. Copy settings: cp .env.example .env.local

2. Install dependencies: npm install

3. Run locally: npm run dev

4. Build: npm run build

DEPLOYMENT STEPS (VERCEL)

Heritage is optimized for Vercel deployment:

  1. Push your fork/clone of the dashboard to GitHub.
  2. Import the project into Vercel dashboard.
  3. Configure the required Environment variables.
  4. Click **Deploy**. Continuous delivery builds updates automatically on pushes to `main`.
